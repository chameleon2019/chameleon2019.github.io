MySQL

SQL: Structured Query Language

程序员和DBA可以完成如下工作：

  1. 在数据库中检索信息
  2. 对数据库的信息进行更新
  3. 改变数据库的结构
  4. 更改系统的安全设置
  5. 增加或回收用户对数据库、表的许可权限

程序员前三个，DBA后两个。

标准的SQL语句通常分为如下几个类型：

  1. 查询语句：select语句

  2. DML（Data Manipulation Language）数据操作语言：insert, update, delete, select

  3. DDL（Data Definition Language）数据定义语言：create, alter, drop, truncate

  4. DCL（Data Control Language）数据控制语言：grant, revoke

  5. 事务控制语句：commit, rollback, savepoint

数据库约束：通过约束可以更好的保证数据表里数据的完整性，约束是在表上强制执行的数据校验规则，主要用于保证数据库里的数据的完整性，除此之外，当表中数据存在相互依赖的时候，可以保护相关的数据不被删除。

大部分数据库支持下面五种完整性约束。

NOT NULL：非空约束，指定某列不能为空；not null

UNIQUE：唯一约束，指定某列或者几列组合不能重复；[constaint 约束名] 约束定义

PRIMARY KEY：主键，指定该列的值可以唯一地标识该条记录；相当于非空约束和唯一约束；primary key，auto_increment用于设置自增长；

FOREIGN KEY：外键，指定该行记录从属于主表中的一条记录，主要用于保证参照的完整性；

CHECK：检查，指定一个布尔表达式，用于指定对应列的值必须满足该表达式。

MySQL不支持CHECK约束。

约束也是数据库对象，也拥有自己的名字，根据约束对数据列的限制，分为两类：单列约束和多列约束。

为数据表建立约束有两个时机：建表的同时为相应的数据列指定约束和建表后创建，以修改表的方式来增加约束；

约束通常都无法修改；MySQL使用information_schema数据库里的TABLE_CONSTRAINTS表来保存该数据库实例中的所有的约束信息。
