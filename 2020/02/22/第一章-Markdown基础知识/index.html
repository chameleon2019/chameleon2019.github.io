第一章-Markdown基础知识

Markdown起源

GFM: GitHub Flavored Markdown
Markdown的灵感来自于纯文本电子邮件的格式，使用简单的纯文本格式细写作，并且可以转成HTML文档。
CommonMark是标准语法，GFM也遵循了CommonMark规范。

为什么使用Markdown

  • 简单：专注于写作，避免时间浪费在排版上
  • 兼容：类似java，一次编写，全平台运行一致

Markdown使用

学习Markdown语法：

  • 基础语法：能写出来（放之四海而皆准的基础语法）
  • 扩展语法：能写的更多（表格、任务列表、删除线、Emoji等）
  • 写作规范：能写的更好（最佳实践）
