Markdown语法

Posted on 2020-02-23

基础语法

写标题

使用两种标记：-/= 或者 #。

-/=：如字面意思，只能支持这两级标题，-是二级标题，=是一级标题

#：支持6级标题

写标题的注意点汇总：

  • 标题尽量简短，方便应用
  • 保持间距，标题前后空一行
  • 不写多余空格

粗体和斜体

粗体：两个*或者两个_包裹

斜体：一个*或者一个_包括

eg:

粗体1 粗体2

斜体1 _斜体2_

注意点：

  • 使用*比_有可读性
  • 粗体和斜体标记内部，建议不用空格

段落和换行

段落：行与行之间要有空行（空行：行内什么都没有，或者只有空格、制表符），
如果想在段内换行，需要在上一行插入两个以上的空格然后按回车键

列表：

有序列表：数字序号 + 英文句号 + 空格 + 内容。eg:

  1. item1
  2. item2
  3. item3

无序列表： */+/- + 空格 + 内容。eg:

  • item
  • item
  • item

注意点：

  • 列表可以嵌套列表
  • 建议使用-来标记无序列表，因为*容易和粗体或者斜体混淆
  • 如果列表项有换行，建议给无序列表使用3个空格，给有序列表使用两个空格

分割线：使用3个以上的 */-/_ 来标记。eg:

Here you are.

what’s your name?

My name is Lilian.

图片

插入图片语法： ! [图片替代文字] （图片地址）。

链接

文字链接：

[链接][url]

[链接][url]

上面这种方式，源文件可读性较差

可以使用下面这种方式：引用链接（把链接地址作为变量在Markdown文件尾部定义好，在正文中引用）

语法说明：

  • 链接标记可以有字母、数字、空格和标点符号
  • 链接标记不区分大小写
  • 定义的链接内容可以放在当前文件的任意位置，建议放页尾
  • 当链接地址为网络地址的时候，要用http/https开头，否则会被识别为本地地址

我们经常使用什么值得买、京东、路由器管理页面

网络链接

将网络地址或者邮箱地址使用<>包裹起来会被自动转换为超链接：

duanzhijie1@huawei.com

代码块

行内代码：`包裹

代码块：Tab键或4个空格开头，建议直接使用围栏代码块代码，并且显式注明语言。

引用

引用使用> + 引用内容来标记

> 引用内容

注意点：

  • 多行引用也可以在每一行的开头都插入> 。
  • 引用中可以嵌套引用。
  • 段落和换行的格式在引用中也适用。

转义

和Python等语言一样，想使用规则中已被使用的特殊符号时，加\进行转义：\\，`，* ，\{}，[]，#

锚点测试

删除线

语法：~~被删除的文字~~

被删除了

表情符号

语法： :表情代码:

😀-:-1:-😭

自动链接

相比基础语法，扩展语法中，url地址可以不用<>包裹

www.baidu.com

表格

  • 单元格用|来分割，建议最前后最后都用|
  • 单元格和|之间的空格会被移除
  • 表头和其他行用-来分割
  • 表格对齐的格式：
    • 左对齐：:
    • 右对齐：-:
    • 居中对齐：:-:
  • 块级元素如代码区块、引用区块不能插入表格中

锚点

书签，使用锚点可以跳转到指定位置，语法如下：

[锚点描述] (#锚点名)

注意事项：

  • 锚点名建议使用字母和数字
  • 区分中英文大写
  • 锚点名中不能包含空格，也不能有特殊字符。
